这是我的第一篇文章

几经波折，终于成功的搭建了一个自己的博客。也不分享什么东西，只是展示一下它搭建成功了。

关于发布文章

1.创建文章

hugo运行后，在命令行里执行

hugo new content/post/"%TITLE%"/index.md
::"%TITLE%"为文章标题

例如我想发布一篇名为《你好》的文章，输入命令

hugo new content/post/"你好"/index.md

然后再到hugo目录下进入content/post/你好/,点击里面的一个（也是唯一一个）index.md文件，开始编辑内容。

可以看到这里是新建了一个文章文件夹，再到文件夹里面新建一个index.md文件，而不是直接新建一个title.md文件，这里是为了插入图片方便，后面会再提到。

2.编辑文章

进入index.md文件后，可以看到如下结构：

其中，红框框住的部分是文章属性，包含作者、日期、标题、分类、标签等信息。绿框框住的部分即为文章正文，用markdown格式编写即可。

==关于文章的图片插入==

markdown的图片插入语法是![pic](/src),其中pic是你想给图片起的名字，即鼠标放上去会显示什么文字，/src即为图片目录。

上文提到了创建文章的时候是新建一个文件夹，再到index.md文件里面编辑文章，这样的好处在于我插入图片的时候可以使用相对路径，直接把图片文件放到文件夹下面，/scr写成该图片的文件名即可。

如果用md编辑器（我用的是Typora）直接插入图片，可能插入的是绝对路径，那么采用绝对路径的图片文章一经发布之后，在文中是无法看到图片的（无论是本地还是线上），这就是为什么我会使用/title/index.md这样的方式来发布文章。

3.上传文章

本地编辑好之后，只能在本地的hugo看到，如果想让文章网上冲浪，就要部署到自己的网站。由于hugo是静态博客，所以对项目的每一步操作完成之后都需要上传到网站。

我这里使用的是github的page页面，每次编辑好新文章之后需要手动push到git仓库。打开命令行，输入如下命令：

git add .
git commit -m "commit"
git branch -M main
git push -u origin main

稍等片刻即可再github上看到自己的文章。

4.半自动化

由于每次发文章都需要用命令行来创建文章，并且还要push到git，于是懒惰的我决定用cmd脚本来实现半自动化。

（1）自动创建文章

在hugo根目录下新建文件newTitle.cmd，编辑并输入以下代码：

@echo off
:: 关闭命令回显
chcp 65001 >nul
:: 设置编码为UTF-8，支持中文输入

:: 提示用户输入文章名称
set /p TITLE=请输入文章名称（支持中文和英文）： 

:: 判断用户是否输入了内容
if "%TITLE%"=="" (
    echo 错误：文章名称不能为空！
    pause
    exit /b
)

:: 执行 hugo new 命令
hugo new content/post/"%TITLE%"/index.md

:: 判断命令是否执行成功
if %errorlevel%==0 (
    echo 成功创建文章：%TITLE%
) else (
    echo 创建文章失败，请检查 Hugo 是否正确安装或路径是否正确。
)

pause

保存后双击运行，命令行会提示输入“文章标题”，输入之后回车，即可在content/post/目录看到新文章的文件夹，开始编辑即可。

（2）自动发布文章

同理，新建文件git_push_all.bat，编辑并输入以下代码：

git add .
git commit -m "commit"
git branch -M main
git push -u origin main
pause

双击运行即可直接将更改过后的项目推送到git仓库，稍等片刻即可在网站上看到新的文章。

5.多端编辑

有时候灵感迸发，或者闲的无聊的时候就会在手机上写文章，如果在手机上写好了再传给电脑多少有点麻烦，于是搭建了一套多端编辑的方案。

（1）坚果云

手机端和电脑端都下载坚果云软件，并将content/post/文件夹在坚果云中设置为同步文件夹，这样即可在手机、平板和电脑上共同访问post文件夹了。

例如我想在手机上写一篇标题为《我好帅》的文章，打开坚果云后在/post云文件夹下面新建文档，此时新建的文档为txt文件，如果不做格式要求，可以直接在txt里面编写，保存之后即可在电脑上看到编写的文字，再重复上述发布文章的步骤即可。

（2）MT文件编辑器

在手机上用txt文件编写markdown语法的文章多少有点不方便，这里我选择用MT文件管理器自带的MT文件编辑器来编辑文章。按照上述方法创建好新文档之后，先保存空文档（**一定要先保存！！！如果编辑之后再保存有几率会丢失！！！**别问我怎么知道的😢）。确认好该文档被保存之后，再到坚果云软件里面点进去，用MT文件编辑器来打开，再用markdown语法编写文章即可，后续步骤与上面相同。
