📦 前置准备清单

  • 一个能收验证码的手机号
  • 1G以上存储空间（给AI小可爱腾个窝）
  • 速溶咖啡☕（防止你在实名认证环节睡着）

🔧 六步驯服AI小怪兽

1️⃣ 火山引擎注册（大型实名认证真人秀）

认证小贴士

  • 企业用户 → 公司注册
  • 学生党 → 个人认证
  • 完成认证后记得领取50万token新人礼包！
  • 记得注册额后先实名认证

访问火山方舟官网，点击"立即体验"

2️⃣ 创建推理接入点（给AI修个高速公路）

接入点创建

  • 点击左侧菜单栏

  • 选择火山方舟大模型产品

  • 在左侧菜单栏中选择在线推理

  • 选择自定义推理接入点 →创建推理接入点

  • 接入点名称随便填，模型选择DeepSeek R1(如果你还想用V3也别急，后面再按相同的方式添加一个V3接入点即可)

  • 如果选择模型提示你未开通的话，按照提示开通一下就好，购买方式选择按Token付费

    保存成功后即可在接入点列表中看到新建的模型：

3️⃣ 获取API密钥（拿到魔法咒语）

密钥获取路径

  • 点击左侧菜单栏
  • 选择API Key管理（在最下面，往下滑才能看到）
  • 点击创建API Key，随便输入一个名称，创建成功后立马复制并找个地方保存下来

重要提示：千万不要随便给出你的秘钥，这就是你家钥匙！！(血泪教训)

4️⃣ 下载Cherry Studio（AI启动器）

在Cherry Studio官网下载对应的Cherry Studio客户端并且安装

5️⃣ 配置连接（人机握手协议）

  1. 点击左下角⚙️ → 模型服务
  2. 下滑找到豆包（火山引擎是字节跳动的产品，所以这里是豆包也没什么问题罢xd）
  3. 填写刚才复制的API秘钥（悄咪咪的，不要让其他人看到你的秘钥）
  4. 重点来了，点击添加模型，这里的模型ID是刚才创建接入点模型下面的那一小串英文！！！，通常是ep-xxxx，点击复制进来
  5. 模型名称写个有辨识度的，比如你是deepseek R1模型就可以写R1，deepseek V3模型就可以写V3（它默认是模型ID的名称，并且新建了就不能更改了😠）
  6. 添加成功后在API秘钥那里点击检查，选择你刚才创建的模型，出现绿色对钩之后就可以了。
  7. 接下来去首页享受你的AI机器人吧~

6️⃣ 提示词工程（让AI听懂人话）

你的专属智能体

Cherry Studio内置了许多助手（智能体），可以在左侧智能体⭐那里选择​，选择你喜欢的智能体，点击添加到助手，再回到聊天页面，选择这个助手开启新的聊天即可

另外也可以自己写一个智能体

  • 点击创建智能体
  • 去一个好听的名称（难听的也行）
  • 输入提示词（格式参考下面的）
```system
你是个精通技术文档的助手，回答时要：
1. 先给出关键结论
2. 用Markdown代码块展示操作步骤
3. 最后补充注意事项
```

然后就可以享受你自己创建的智能体了（我才不会告诉你这里可以创建一个猫娘角色和你对话）

  ```bash
🌈 效果展示 

```
# 用户
用吃火锅解释Transformer架构

# DeepSeek R1
1. **顾客点单（输入嵌入）** 

- 每位顾客（词语）带着自己的喜好（词向量）进店，比如牛肉党、青菜党、海鲜党
- 服务员用扫码枪把菜品转化为数字订单（嵌入向量）

2. **智能服务员（自注意力机制）** 

- 服务员会观察全场顾客的关联："穿红衣服的女士点了毛肚，她旁边的孩子可能需要儿童套餐"
- 通过电子手环（位置编码）记住每桌的座位顺序
- 后厨大屏实时显示各桌需求关联强度（注意力权重矩阵）

3. **多岗位协作（多头注意力）** 

- 食材调度员关注荤素搭配比例
- 火候管理员监控不同汤底的沸腾程度
- 传菜组长协调上菜顺序优先级
- 8个岗位（8个头）各自专注不同维度

4. **中央厨房（前馈网络）** 

- 收到所有需求后，中央厨房开始统一处理：
- 切肉机处理肉类订单（矩阵变换）
- 洗菜流水线处理蔬菜订单（非线性激活）
- 打包台统一封装（输出向量）

5. **智能调度系统（残差连接 & 层归一化）** 

- 自动调节灶台火力：太咸的锅底加点汤（残差连接）
- 平衡不同区域的温度（层归一化）
- 确保每轮加工保持稳定状态

6. **上菜机器人（解码器）** 

- 根据累积的订单信息逐步输出：
- 先上锅底（起始符）
- 接着按顺序摆放食材（生成序列）
- 实时调整摆盘方式（掩码注意力）
- 直到摆满餐桌（遇到终止符）

7. **顾客反馈（损失计算）** 

- 顾客评分系统对比：
- 预期摆盘（标注数据）vs 实际上菜（预测结果）
- 调整后厨参数让下次摆盘更符合预期

整个过程就像吃火锅时的动态协作：每个食材的处理既考虑自身特性，也关注其他食材的搭配，通过多轮协调最终呈现完美的火锅盛宴。这种并行处理方式，正是Transformer比RNN更快更高效的关键所在。
